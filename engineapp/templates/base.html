The Web

The web is a collection of digital files, hosted on multiple servers. Web-Browsers (Internet Explorer, Firefox, Chrome etc) are the most commonly used GUI interfaces for navigating (surfing) the web.

HTTP: is the most common protocol used by web-browsers to request specific files from the multitude of servers around the world.

HTML: is the standard language used to create web pages. There are reportedly over 30 billion pages/documents out there. HTML Documents, and their related structures, can be analyzed using the following parameters;

  • The Doctype - What kind of HTML this is.
  • The head includes metadata, CSS and Java, and the title of the page, which appears on the top of the browser window.
  • The body - is the actual contents of the document.
  • Text content - what you see.
  • Referecnes to other docs; e.g videos, images etc.
  • Links to other pages.
  • Markup - what it looks like. Also called 'tags', these are machine instructions for the browser to interprete/present the files retrieved from the remote servers to the end-user/web-surfer in the manner envisioned by the creator/originator of the HTML document.
  • The opening and closing HTML tags surround elements of the document.

Tags and Elements

An HTML tag is always contained within angled brackets. "void" tags do not require a closing tag (e.g. <br>). Tags, and their related 'elements' [ie values within a set of opening and closing tags] can be categorized into; Inline and Block elements.

Inline elements: <b>, <em>, <a>, <br>, <span>, <img>,
Block elements: <p>, <div>, <form>.

Some of these elements are described below, along with possible uses;

  • bold : <b>value</b>
  • italics : <em>value</em>. or <em><b>value</b></em> [tag within a tag].
  • ancor : <a href="url">value</a> eg This website is <a href="www.udacity.com">my favorite</a>.
  • image : <img> does not need closing tag [a type of Void tag]. Always add 'alternate text' in case image is missing/broken. eg <img src="url" alt="sorry">
  • break : <br> for page break. [also a type of Void tag].
  • paragraph : <p> value </p>

Some helpful tips:

Here are some helpful tips for writing HTML code;

  • Close the tag element syntax [ie </b>] otherwise everything after will have the same properties.
  • It's advisable to type both opening AND closing tag at the start of the line of code.
  • Whitespace assumes everything is on one line. Tag elements are needed to force breaks, paragraphs, etc.
  • Block elements create "invisible boxes".
  • Attribute : This is a property of an HTML element.

Lesson 3: Adding CSS Style to HTML Structure

CSS [Cascading Style Sheets]

CSS allows programmers to bring style to plain html documents. This programming language, not only bring style to plain html documents, but also to minimize repetition while writing code through use of selectors and other functionality.

Selectors

These are the biggest key to understanding CSS. They define what elements of a page a particular style will apply to. They allow you to target specific HTML elements and apply style to them.

-Here's a quick example;

h1 {
background-color : black; color: white;
}

In the above sample syntax;

-h1 is the selector.

-Everything between these curly braces will be interpreted as a "rule" that should be applied to every h1 element.

-The line of code that says background-color : black; is a rule.

-'background-color' is an attribute and black is the value of that attribute.

Including CSS Styling;

There several methods to include CSS styling in your web page. Here are three such methods.

Method 1: write CSS within the <head> of your HTML. ie At the top of your HTML document, add a <head> element and write the style syntax within it.

Method 2: Link your HTML to a separate CSS file [NOTE: Both files MUST be in the same folder].

Method 3: Write your style inline with your HTML. ie Inside the html <body>. This is generally a very bad idea because it leads to lots of repeated code. But you may see code that uses this method so it's good to be familiar with it.

Lesson 2: Variables and Strings

Variables in Python

Variables are used to create a name which then refers to a value. This allows for the use of the same expressions, and by only changing the values, to then compute many diff things. This also makes program code a lot easier to understand, even in the abstract.

For example;

If my_variable is assigned a value of 4, then the following code would print out 8:

print my_variable + my_variable

Some benefits of variables include:

  • They improve code readability by using names that make sense to humans.
  • They give us a way to store the value of important data.
  • Assigned values can easily be changed as needed.

Note : Name refers to the value of the expression. The name can be any sequreence of numbers, letters and underscores, as long as it starts with a letter or underscore.

Assignment Statement

This is a way to introduce a variable, ie Assign value, to a variable.

For example;

We can assign the value 4 to the variable my_variable with the following code:
my_variable = 4

Equals sign [=] in Python

In Python, the equals sign [=] means 'assignment'. As opposed to the traditional equals seen in algebra. It can also be seen as an arrow pointing to the expression assigned to (or defining) the variable/name.

For example;

In the line 2 + 3 = 5, the equals sign means "is the same as".

In the line my_variable = 5, the equals sign means "takes the value of".

Strings

Strings are a sequence of characters surrounded by quotes. Single (') or double quotes (") can be used, as long as you start and end the string with the same type of quote. ie The quotes tell the computer the expression is not a variable. Strings can also be assigned to variables, if desired.

For example;

In Python, 2 is a number while "2" is a string.

The code 2 + 2 would give 4.

The code "2" + "2" would give "22".

Note : In addition to using single or double quotes to create a string, you can also use TRIPLE quotes (''') to create a multi-line string.

Lesson 3: Input -> Function -> Output

Function

A function takes input, does something to that input, and then produces output. For example, a function named sum might take a numbers as input and produce the sum of those numbers as output. A function is sometimes refered to as a "procedure").

Making vs. Using Functions

A function line of code starts with keyword def. Then give the function a name followed by the function parameters inside parentheses. These parameters will be replaced in the output by actual values when the function is used (called).

In the "body" of the function, we specify what to do with the input parameters.

For example;

def sum(a, b):
sum = a + b
return sum

To use a function, we write the name of the function followed by the value(s) we want to give it in parentheses. Like this:

print sum(2, 10)
>>>12

Avoiding Repetition by using Functions

Define a function once, and you never have to define it again. Programmers can create functions and reuse those function and definitions indefinitely. This helps greately in avoiding repetion.

Function without a Return statement?

return keyword specifies what output the input should produce. Observe the two output behaviours below;

First without a return statement;

def add_two(x):
answer = x + 2
new_number = add_two(7)
print new_number
>>>None

Now with a return statemnent added;

def add_two(x):
answer = x + 2
return answer
new_number = add_two(7)
print new_number
>>>9

Lesson 4: Decisions and Repetition - If & While

Comparisons

Comparisons are the foundation of if statements. Comparison operators [ '<' '>' '==' '!=' ] are used to execute the comparison function. Comparisons generate output in form of Boolean value [ True or False ].

Here are some examples of comparison statements;

print 2 > 1
>>>True
print 1 > 2
>>>False
print 7 * 3 < 21
>>>False
print 7 * 3 != 21
>>>False
print 7 * 3 == 21
>>>True
print "hello" == "hello"
>>>True

>>>False

>>>False

if

An if statement is used to make decisions. if modifies the behavior of the function based on what the input is, thus making your code do something different depending on the result of comparisons. This is because if statements test the outcomes of comparisons to generating the desired output.

Below is a simple if procedure;

def bigger(a, b):
if a < b:
return b
return a

print bigger(2,7)
>>>7
print bigger(3,2)
>>>3
print bigger(3,3)
>>>3

else

else is part of an if statement. Though used in a separate block, it is still within the structure the if function.

Below is an if procedure with an else statement;

def is_friend(name):
if name[0] == 'D':
return True
else:
return False

print is_friend('Diane')
>>>True
print is_friend('Fred')
>>>False

Note : It is important to note close attention should be paid to the placement of indentations, as these are VERY important to the proper execution of an if statement.

While

The while statement is a loop construct in Python. This function is very useful in writing code, and helps reduce repetition. A while loop repeatedly executes the body of the loop until the test condition is no longer True. In other words; it will continue performing the function[ie executing the test expression within the block] as long as the test expression returns True. When the test expression returns False, the execution moves on to the next expression in the next block.

Below shows a while loop;

counter = 0
while counter != 4:
counter = counter + 1
print counter

>>>1
>>>2
>>>3
>>>4

The output from this procedure stops at 4 because 4 != 4 is False [i.e. print 4 != 4 >>> False ]

Below is an infinite while loop;

counter = 1
while counter != 4:
counter = counter + 2
print counter

Since the value of counter will never equal 4, the loop test is never False [or is always True]. That means the loop runs forever. The loop will keep executing as long as the loop test evaluates to a True value. In this case, it will always be True .

Mutation

Mutation is when the elements of an object are changed in one form or another. This gives rise to the terms mutability and immutability. Certain mutations will necessarily effect the original intended output from the variables within the code which refer to the modified object.

Mutability vs Immutability

Mutable objects within a list or string can be changed. Immutable strings are not affected by this phenomenon. This is because 'string' objects do not support item assignments, so you cannot change an item within a string object.
Below are some simple mutations;

Mutation List:

p = ['H','e','l','l','o']
print p
>>>['H','e','l','l','o']

p[0] = 'Y'
print p
>>>['Y','e','l','l','o']

Mutation String:

s='Hello'
print s
>>>'Hello'

s ='Yello'
print s
>>>'Yello'

s = s + 'w'
print s
>>>'Yellow'

Immutable string object:

s = 'Hello'
s[0] = 'Y'
>>>'Error: item assignment'

Aliasing

Aliasing is when there are two different ways to refer to the same object. Any changes made to the object will effect whatever variables refer to that same object. However, if one of the variables pointing to the same object is assigned a new value [object], that will not affect the remaining variable [ie the remaining variable will still refer to the unmodified/original object]. Unless, of course, the original object is also modified.

For example;
If p and q both refer to ['J','a','m','e','s'].
Apply p[2] = 'n'
Now p refers to new object ['J','a','n','e','s'], while q still refers to ['J','a','m','e','s'].

An Object-oriented programming language relies heavily on the programming concept of "objects". These object data structures have specific attributes and procedures/methods. Python is an Object-oriented programming language that makes use of not only objects, but also contructs called classes, and modules, that allow for the structuring of your software. Lesson 1: Networks

Networks

A network is a group, with a minimum of three entities, that can communicate, even though they are not directly connected. Human networks go back at least three thousand years.

To make a network, we need a way to encode and send a message. A way to route the message, and finally, on the receiving end, a way to recieve and decode the message. In cases of multiple messages being sent simultaniously, we also need a way to priorities messages.

Networks are measured by Latancy (Time it takes mesage to travel from sender to receiver as measured in unit of time) and Bandwidth (Amount of data that can be sent in a unit of time as measured in bits per second).

Protocols

These are a set of rules that govern how two or more entities communicate with each other. HTTP is a type of protocol.

URL

A URL (Uniform Resource Locator) is one type of Uniform Resource Identifier (URI). A URL is a reference to a resource that specifies the location of the resource on a computer network, or on the world wide web, along with a mechanism for retrieving it.

Here are two types of parameters that can be deployed to acheive this end;

GET Parameter : This is a query parameter used to request documents from a server.

POST Parameter : This is a parameter used to send documents to server.

Query Parameters (GET parameter) is usually seperated from a url by the symbol

'?'
. We can string multiple queries together with the symbol
'&'
.

For example;

http://....com/foo?p=1&q=neat

caching

A cache (pronounced like cash) is something that stores data locally so that you don't have to retrieve it later from a remote server. It can be used to make data requests faster.

HTTP Server Responses

HTTP server responses are what is generated by the sever whenever a request is made to the server. Below are two types of HTTP server responses;

Static : These are pre-written files and images stored on the server.

Dynamic : These made on the fly (ie modified based on the input request). Web applications are also dynamic.

XML

XML (Extensible Markup Language) is a text-based format that allows for the structuring of electronic documents and is not limited to a set of labels. XML data is known as self-describing or self-defining. An XML document is made up of nested elements which are defined by tags. An element has a beginning and an ending tag. Elements in XML documents are all contained in an outermost element called the root element. This allows for hierarchical structures. XML also supports defined attributes for elements and describes characteristics of the elements in the beginning tag of an element.

  • parsing xml : Parsing is the act of splitting up information into its component parts. All modern browsers have a built-in XML parser.
  • xml.dom : Dom (Document Object Model) is a computer representation of xml. An XML parser converts an XML document into an XML DOM object.
  • minidom : Simple and fast version of xml.dom
